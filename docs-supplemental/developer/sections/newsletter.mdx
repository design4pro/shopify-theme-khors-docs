## Location

`sections/newsletter.liquid`

## Overview

The newsletter section provides email capture functionality using Shopify's native customer contact form. It includes GDPR-compliant subscription handling, honeypot bot protection, and accessible form validation with success and error states.

## Architecture

This section uses Shopify's `{% form 'customer' %}` tag to create contact submissions that are automatically tagged as "newsletter" for segmentation in Shopify's customer database or third-party email marketing integrations.

| Component | Description |
|-----------|-------------|
| Form Header | Customizable heading and rich text description |
| Email Input | Required email field with validation |
| Submit Button | Renders via `button` snippet |
| Honeypot Field | Hidden field to catch bot submissions |
| Success State | Confirmation message with icon |
| Error State | Form validation error display |

## Form Behavior

### Honeypot Protection

A hidden `contact[website]` URL field is positioned off-screen with `tabindex="-1"` and `autocomplete="off"`. Legitimate users won't interact with it, but bots filling all fields will be caught by Shopify's spam filter.

### Email Validation

- HTML5 `required` and `type="email"` for browser validation
- `aria-required="true"` for screen reader accessibility
- `autocomplete="email"` enables browser autofill

### State Handling

- **Success**: After successful submission, `form.posted_successfully?` displays a confirmation message with `role="status"` for screen readers
- **Error**: `form.errors` displays Shopify's default validation errors with `role="alert"`

### Form Submission

Submits to Shopify's `/contact` endpoint with hidden tag `newsletter`. Emails are stored as customer contacts and can be exported or synced to email marketing platforms via Shopify apps.

## Accessibility

- Visible label hidden with `.visually-hidden` class, present for screen readers
- Email input includes `aria-required="true"`
- Section has `role="region"` with translated `aria-label`
- Success message uses `role="status"` for polite announcement
- Error messages use `role="alert"` for immediate announcement
- Focus state includes 3px outline with accent color
- Honeypot excluded from tab order and hidden from assistive tech

## GDPR Compliance

This section collects email addresses via Shopify's customer contact API. For GDPR compliance:

1. Ensure store's privacy policy includes newsletter subscription disclosure
2. Consider adding checkbox for explicit consent if targeting EU customers
3. Use Shopify's customer data export features for GDPR access requests
4. Implement unsubscribe links in email campaigns

<Note>
Shopify automatically includes customer data in their compliance tools. Consult with legal counsel for specific regional requirements.
</Note>

## Integration

Submitted emails appear in **Shopify Admin > Customers** with tag "newsletter". Common integrations:

- **Klaviyo**: Sync newsletter subscribers to dedicated list
- **Mailchimp**: Export tagged customers or use Mailchimp app
- **Shopify Email**: Create segment for customers with "newsletter" tag
