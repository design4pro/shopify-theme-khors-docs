## Architecture

The search component is a full-screen overlay with three states:

1. **Hidden** â€” Default (`hidden` attribute)
2. **Open with input** â€” Triggered by `[data-header-search-toggle]` buttons
3. **Showing results** â€” After 2+ characters typed (300ms debounce)

### API Integration

Uses the Shopify Predictive Search API:

```
/search/suggest.json?q={query}&resources[type]=product,collection,page,article&resources[limit]=4
```

Results are cached in a `Map` per session. AbortController cancels in-flight requests on new input.

## JavaScript

The `PredictiveSearch` class (inline via `{% javascript %}`) provides:

- **Open/close**: Toggle via `[data-header-search-toggle]`, close via overlay/button/Escape
- **Debounced search**: 300ms delay, minimum 2 characters
- **Result caching**: `Map`-based cache prevents redundant API calls
- **Request cancellation**: `AbortController` aborts previous request on new input
- **Keyboard navigation**: Arrow Up/Down to navigate results, Enter to select
- **Result rendering**: DOM creation for 4 resource types with images and prices
- **Scroll lock**: `scroll-lock` class on body when open
- **Focus management**: Auto-focus input on open, return to toggle on close
- **Private fields**: `#input`, `#results`, `#cache`, `#controller`, `#selectedIndex`, etc.

## Accessibility

- ARIA combobox pattern with `role="combobox"`, `role="listbox"`, `role="option"`
- `aria-expanded`, `aria-autocomplete="list"`, `aria-controls` linkage
- `aria-selected` on active result item
- Visually hidden label for search input
- Escape key closes overlay
- `focus-visible` outlines on all interactive elements
- `prefers-reduced-motion` disables transitions and spinner animation
