## Color Schemes

The Khors theme offers **three color schemes** that you can assign to different sections on the page. Each color scheme contains a complete set of colors for backgrounds, text, accents, and borders.

### Color Variables in Schemes

Each color scheme consists of **6 color variables**:

| Variable | Purpose | Example usage |
|---------|---------------|-----------------|
| **Background** | Main section background | Section or card background |
| **Foreground** | Main text color | Headings, primary text |
| **Background 2** | Secondary background | Card backgrounds within sections, form fields |
| **Foreground 2** | Secondary text | Descriptions, helper text, metadata |
| **Accent** | Accent color | Buttons, links, highlight icons |
| **Border** | Border color | Separators, card borders, inputs |

### How to Change Scheme Colors

1. Open **Shopify Admin > Online Store > Themes**
2. Click **Customize** on your theme
3. In the left panel, click **Theme Settings** (gear icon)
4. Go to the **Colors** section
5. Select the scheme you want to edit (Light, Dark, or Accent)
6. Change colors for each variable:
   - **Background** - main background
   - **Foreground** - main text
   - **Background 2** - secondary background
   - **Foreground 2** - secondary text
   - **Accent** - accent color
   - **Border** - borders
7. Click **Save** to save changes

<Info>
  Changes to the color scheme will automatically update **all sections** using that scheme.
</Info>

### Assigning a Scheme to Sections

Most sections in the Khors theme allow you to choose a color scheme:

1. In the Theme Customizer, click the section you want to edit
2. Find the **"Color scheme"** setting
3. Select one of three schemes:
   - **Light** - light background
   - **Dark** - dark background
   - **Accent** - accent background
4. The preview will change immediately

## Best Practices

### Contrast and Readability

<Warning>
  Ensure that the contrast between text and background is sufficient (min. 4.5:1 for normal text) to meet WCAG 2.1 accessibility standards.
</Warning>

**Contrast checking tools:**
- [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/)
- Chrome DevTools - Lighthouse Accessibility Audit

### Branding Consistency

- Use brand colors in color schemes
- **Light scheme** - use most frequently (80-90% of sections)
- **Dark scheme** - use for contrast (10-15% of sections)
- **Accent scheme** - use sparingly for key CTAs (5% of sections)

### Visual Hierarchy

```
Hero Section (Light)
  ↓
Features (Light)
  ↓
Testimonial (Dark) ← Contrast
  ↓
Products (Light)
  ↓
Newsletter CTA (Accent) ← Attention!
  ↓
Footer (Dark)
```

## Advanced Tips

### Accent Color for Buttons

The **Accent** color in each scheme is used for:
- "Add to Cart" buttons
- Links in text
- Interactive icons
- Hover states

Choose a color that is clearly visible and encourages action.

### Background 2 for Depth

Use **Background 2** to create subtle visual hierarchy:
- Product cards on section backgrounds
- Form fields
- Quotes and testimonials
- Sections within sections

The difference between Background and Background 2 should be subtle (5-10% brightness difference).

### Foreground 2 for Text Hierarchy

**Foreground 2** (secondary text) should be:
- 60-70% opacity relative to Foreground
- Still readable (min. 4.5:1 contrast ratio)
- Used for: prices (crossed out), dates, metadata, short descriptions

## Testing Schemes

Before finalizing color schemes:

1. **Check on mobile devices** - colors look different on small screens
2. **Test in different lighting** - bright sunlight vs. dark room
3. **Check accessibility** - use Lighthouse Audit in Chrome DevTools
4. **Compare with competitors** - ensure you stand out positively
5. **Gather feedback** - ask your team/customers for opinions
